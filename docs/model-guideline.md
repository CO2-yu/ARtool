# モデル作成ガイド

## 前提

本システムは通常3DビューとARビューの両方で同じGLBを使う。Viewerでは `model-viewer`、ARではThree.js `GLTFLoader` が読み込むため、両方で安定して扱えるGLBを作成する。

## Rhino運用

Rhino側はmm運用を許容する。ただし、GLB出力後にアプリ側のThree.js空間へ合わせるため、`package.json` の `transform.scale` と `scale.default` で補正できるようにする。

## RhinoからGLBへの推奨フロー

1. Rhino上で実寸単位を確認する。
2. 不要な微細ディテールを削除する。
3. 表示用メッシュを生成する。
4. 原点と向きを整理する。
5. GLBとして出力する。
6. `/viewer` で見た目と操作性を確認する。
7. `/ar` でマーカー上のスケールと追従安定性を確認する。

## 軽量化ルール

以下は原則として簡略化する。

- ボルト
- ナット
- ネジ山
- 小穴
- 微細金具
- 裏面ディテール

製造詳細よりも、展示会で製品形状と特徴が伝わることを優先する。

## ネット表現

実メッシュ網は禁止推奨とする。ネットや格子はサーフェス + テクスチャ、またはアルファ表現を使う。

## Draco圧縮

Draco圧縮は初期MVPでは非推奨とする。

理由:

- デコーダ配置が必要
- 初回ロード時のCPU負荷が増える
- 端末差の検証が必要

使う場合は `public/draco/` にデコーダを配置し、Three.js側で `DRACOLoader` を設定する。

## 座標系

RhinoのZ-upからglTFのY-upへ変換される点に注意する。出力後、以下を確認する。

- `/viewer` で正しい向きに見えるか
- `/ar` でマーカーに対して自然な向きか
- 必要な固定補正は `transform.rotation` へ入れる

## 原点ルール

- 製品中心またはマーカー上に置きたい基準点を原点付近に置く。
- 遠方にCAD由来の不要オブジェクトを残さない。
- AR表示時にマーカー平面から大きく外れないようにする。

## メッシュ品質

Rhinoのメッシュ品質は、見た目と軽量性のバランスを優先する。

推奨:

- 曲面は過剰分割しない。
- 平面はできるだけ少ないポリゴンにする。
- 見えない裏側や内部は削除する。
- 小さな丸みや穴はテクスチャまたは省略で代替する。

## テクスチャ

- 画像サイズは必要最小限にする。
- 共有可能なマテリアルはまとめる。
- 工業製品の単色面に過大なテクスチャを使わない。
- 透過テクスチャは実機で描画順と負荷を確認する。

## 検証チェックリスト

- `/viewer` でGLBが読み込める。
- `/ar` でGLBが読み込める。
- 実寸スケールが正しい。
- `scale.default` で展示会向けサイズになる。
- 原点と回転が正しい。
- 24fps以上を維持できる。
- スマートフォン実機でロード時間が許容範囲内である。
