# 要件定義

## 目的

本システムは、展示会および将来的な営業用途に向けたスマートフォン用3D/ARビューアである。QRコードから起動した来場者に対し、まずAR.jsのカメラ画面を表示し、マーカー認識後にGLBをAR表示する。`model-viewer` は起動画面ではなく、マーカー認識後に選択できる補助的な通常3Dビューとする。

初期MVPでは、展示会場でARマーカー体験を成立させることを最優先する。AR表示後に、必要に応じて `model-viewer` を使う通常3Dビューへ遷移できる。

## 画面構成

本システムは以下の2画面を持つ。

- `/` または `/ar`: マーカーARビュー
- `/viewer`: 通常3Dビュー

ARビューがメイン入口である。Viewerはマーカー認識後に選択される補助ビューであり、`package` query parameter で対象パッケージを受け取る。

例:

- `/viewer?package=fence_a`
- `/ar?package=fence_a`

`package` が未指定または不正な場合は、パッケージインデックスの先頭パッケージを既定値として扱う。

## 通常3Dビュー要件

通常3Dビューは `model-viewer` を中心UIとする。

目的:

- 安定した3D閲覧
- スマートフォンでの自然な操作
- AR非対応またはARが不安定な端末でも成立する表示
- AR表示後に詳細確認する補助導線
- 将来的な営業ツールへの拡張候補

役割:

- GLB表示
- ピンチ操作
- 回転
- 拡大縮小
- 慣性操作
- アニメーション再生
- スケール変更
- 製品説明表示
- ARビューから遷移した対象製品の通常3D表示

## マーカーARビュー要件

マーカーARビューは既存のAR.js + Three.js構成を維持し、アプリのメイン入口とする。

AR.jsが担当すること:

- markerId認識
- packageId特定
- AR空間への配置
- GLBのAR表示
- `model-viewer` への遷移UI表示

マーカー未認識時は `model-viewer` 遷移UIを表示しない。マーカー認識後、AR表示と同時に「3Dビューで見る」ボタンを有効化する。

## UXフロー

基本フロー:

1. QRコードを読み取る。
2. AR.jsのカメラ画面を開く。
3. マーカーを認識する。
4. 従来どおりマーカー上にGLBモデルをAR表示する。
5. 同時に画面下部へ「3Dビューで見る」ボタンを表示する。
6. ユーザーが押した場合のみ `/viewer?package=xxx` へ遷移する。

## 情報パネル要件

ARマーカー認識後、画面下部へ製品情報パネルを表示する。パネルはAR表示を妨げない補助UIとする。

表示内容:

- 製品名
- 簡易説明
- 「3Dビューで見る」

「3Dビューで見る」押下時:

- `/viewer?package=xxx` へ遷移する。

AR表示自体はマーカー認識時に自動で行う。

## 複数マーカー要件

AR空間:

- 最大3モデル表示
- 上限超過時は新規認識を無視する。
- 既存モデルを勝手に消さない。

操作パネル:

- 表示する情報パネルは1つのみ。
- 最後に認識した `markerId` の `packageId` を `selectedPackageId` とする。
- 情報パネルは `selectedPackageId` の内容で上書きする。
- 「3Dビューで見る」ボタンは `selectedPackageId` を対象にする。

## スケール要件

通常3Dビュー:

- `model-viewer` ベースでピンチイン/アウトに対応する。
- スライダーUIでスケール変更できる。
- 初期値、最小値、最大値、刻み幅は `package.json` の `scale` を使う。

ARビュー:

- MVPでは `package.json` の `scale.default` を使用する。
- 将来的にAR側でもスケール変更できる構造を維持する。

## パッケージ要件

`package.json` は通常3DビューとARビュー共通の単一データソースとする。

共有する情報:

- `model.path`
- `name`
- `description`
- `scale`
- `animation`
- `ui.thumbnail`
- `transform`

追加UI設定:

- `ui.enablePinchScale`
- `ui.showScaleSlider`

## エラー方針

通常ユーザー向けには短く非技術的なエラーのみ表示する。

表示禁止:

- stack trace
- WebGL内部詳細
- 内部例外の詳細

開発者向け詳細は `console.error` とデバッグモードに出す。

## デバッグ要件

通常ユーザーには詳細を表示しない。

デバッグモードは以下の場合のみ有効化する。

- debug専用URL
- `?debug=1`
- `localStorage` flag

## パフォーマンス要件

通常3D閲覧:

- 描画と操作の基本機能は `model-viewer` に委譲する。

AR:

- マーカー追従安定性を優先する。
- 24fps以上を目標とする。
- 最大3モデル表示とする。
- 設定、パッケージ、モデルのロード段階を分離する。

## 受け入れ条件

- `/viewer?package=xxx` で通常3Dビューが開く。
- `/` または `/ar` でARビューが開く。
- `package.json` が両画面の単一データソースとして使われる。
- 通常3DビューでGLBを閲覧できる。
- ARビューでマーカー認識後にGLBをAR表示できる。
- ARビューでマーカー認識後に「3Dビューで見る」ボタンが出る。
- 情報パネルから通常3Dビューへ遷移できる。
- AR空間では最大3モデルまで表示できる。
- 操作パネルは最後に認識した製品の内容だけを表示する。
